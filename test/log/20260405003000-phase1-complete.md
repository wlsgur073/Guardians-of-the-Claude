## Cross-Framework Patterns Discovered

### Pattern 1: Pre-generation scan prevents hallucination (validated)
Phase 2.5S eliminated all path/dependency hallucination across 16 scenarios and 7 languages. Zero instances of referencing non-existent directories or uninstalled packages after the improvement.

### Pattern 2: Existing projects need conciseness discipline
Batch 2 existing-state scenarios scored 4.8 due to prescriptive suggestions ("consider adding ruff", "you should set up ESLint"). After adding the "Document what IS" rule, Batch 3 existing-state scenarios all scored 5.0.

### Pattern 3: .gitignore handling was ambiguous
The original instruction "append if not already present" didn't cover the case where .gitignore doesn't exist. Fixed to explicitly handle both cases.

### Pattern 4: Monorepo is genuinely harder
The only scenario that scored below 5.0 in Batch 3. Root vs per-package command separation, multi-framework detection, and settings.json completeness are unique challenges. The starter template may need monorepo-specific guidance.

### Pattern 5: Agent safety — Write tool can overwrite repo files
One agent accidentally overwrote the repo's CLAUDE.md. Mitigated by instructing agents to use Bash for /tmp writes only.
