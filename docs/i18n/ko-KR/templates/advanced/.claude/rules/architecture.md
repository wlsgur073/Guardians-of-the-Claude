# 아키텍처

## 레이어 구조

위에서 아래 순서:

1. **API 핸들러** (`src/api/`) -- HTTP 요청/응답, 입력 검증
2. **서비스** (`src/services/`) -- 비즈니스 로직, 오케스트레이션
3. **레포지토리** (`src/repos/`) -- 데이터베이스 접근, SQL 쿼리
4. **모델** (`src/models/`) -- Zod 스키마, TypeScript 인터페이스

## 의존성 방향

- 핸들러는 서비스를 호출 -- 레포지토리나 모델을 직접 호출하지 않음
- 서비스는 레포지토리를 호출 -- 데이터베이스에 직접 접근하지 않음
- 레포지토리는 타입 안전성을 위해 모델을 사용
- 레이어 간 순환 의존성 금지
- `src/utils/`의 공유 유틸리티는 모든 레이어에서 import 가능

## 모듈 경계

- 각 엔티티(Task, User, Comment)는 각 레이어에 자체 파일을 가짐
- 엔티티 간 호출은 서비스를 통할 것, 레포지토리 직접 접근 금지
- `src/errors/` 디렉토리는 공유 인프라 -- 모든 레이어에서 import 가능
