# 단계

## 단계 1: 인자 파싱

`$ARGUMENTS`를 읽습니다. `--fix`가 포함되어 있으면 자동 수정 모드로 린트를 실행합니다.

## 단계 2: 빌드

`npm run build`를 실행하여 TypeScript가 오류 없이 컴파일되는지 확인합니다.
빌드가 실패하면 오류를 보고하고 중단합니다 — 린트나 테스트로 진행하지 않습니다.

## 단계 3: 린트

`npm run lint`를 실행합니다 (`--fix`가 요청된 경우 `npm run lint -- --fix`).
남아 있는 경고 또는 오류를 보고합니다.

## 단계 4: 테스트

`npm test`를 실행하여 전체 테스트 스위트를 실행합니다.
실패한 항목의 테스트 이름과 오류 메시지를 보고합니다.

## 단계 5: 요약

요약 테이블을 출력합니다:

| 검사 | 상태 |
| ---- | ---- |
| Build | pass / fail |
| Lint  | pass / fail (N warnings) |
| Test  | pass / fail (N/M passed) |

모든 검사가 통과하면 프로젝트가 커밋 준비가 되었음을 확인합니다.
검사가 하나라도 실패하면 오류 출력을 기반으로 가장 가능성 높은 수정 방법을 제안합니다.
